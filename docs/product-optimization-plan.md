## Sprint 1: 信号焦点 + 评分可解释性 (P0 — 用户价值最高)

**目标:** 解决"打开满屏数据，找不到关键信号"的核心痛点

### 1.1 信号焦点面板 (SignalFocus)

**问题:** 7 个卡片同时展示，用户需要在噪音中手动找共振信号
**方案:** 新增 `SignalFocus` 卡片，放在 dashboard 最顶部，只展示**多维度共振**的标的

共振条件 (满足 ≥3 项即入选):
- 在收筹池 (inPool)
- OI 6h 变化 > 3% (|d6h| > 3)
- 负费率 (frPct < -0.01)
- CoinGecko Trending (inCG)
- 叙事关联 (有对应链上叙事簇)
- 成交量突增 (volSurge)

**实施:**
- [ ] `src/components/cards/SignalFocus.tsx` — 新卡片
- [ ] `src/logic/resonance.ts` — 共振检测逻辑
- [ ] 更新 `Dashboard.tsx` 布局，SignalFocus 占满一行 (w:12, h:4)
- [ ] 每个信号标的展示: 币名 + 共振因子标签 + 综合分 + 一句话解释
- [ ] 点击可打开 DetailDrawer

### 1.2 评分可解释性

**问题:** 用户看到 score=75 但不知道为什么
**方案:** DetailDrawer 增加评分雷达图 + 因子拆解

**实施:**
- [ ] `src/components/shared/RadarChart.tsx` — 四维雷达图组件 (费率/市值/横盘/OI)
- [ ] DetailDrawer 中增加"评分拆解"区域:
  - 雷达图可视化
  - 每个因子的具体数值和得分 (fSc/mSc/sSc/oSc)
  - 一句话解释: "高分原因: 负费率-0.05% + 小市值$32M + 横盘67天"
- [ ] CombinedScore 和 AmbushCandidate 的行内增加 tooltip 显示因子明细

### 1.3 StatusBar 信号滚动条

**问题:** StatusBar 只显示市场概况，没有"今日关键信号"提示
**方案:** 在 StatusBar 增加信号滚动条

**实施:**
- [ ] StatusBar 增加"今日信号"区域，滚动展示最新共振信号
- [ ] 格式: "🔥 XXX 4维共振 78分" + 点击跳转到 DetailDrawer
- [ ] 数据来源: 复用 SignalFocus 的共振检测结果

## Sprint 2: 通知告警系统 (P0 — 时效性是交易工具的生命线)

**目标:** 用户不需要一直盯着屏幕，关键信号自动推送

### 2.1 浏览器通知

**实施:**
- [ ] `src/logic/notifications.ts` — 通知管理模块
- [ ] 首次加载时请求 Notification 权限 (非侵入式，在用户主动点击"开启通知"后)
- [ ] 触发条件 (可配置):
  - 新的共振信号出现 (SignalFocus 新增标的)
  - 收筹池状态从 warming → firing
  - OI 突增 > 10%
  - 链上叙事出现 3★ 新币

- [ ] 防抖: 同一标的 30min 内不重复通知

## 技术债务清理 (穿插在各 Sprint 中)

- [ ] **类型安全:** hooks.ts 中 `Record<string, any>` 替换为具体类型
- [ ] **错误处理:** Binance API 全部 try-catch 但只返回 null，应增加重试逻辑
- [ ] **性能:** 收筹分析遍历所有 ~200 合约的 180 天 K 线，考虑增量计算
- [ ] **响应式:** grid layout 在所有断点都用同一套 layout，应该按断点定义不同默认布局
- [ ] **国际化:** 硬编码中英混杂，统一为中文 (目标用户是中国交易者)
- [ ] **无障碍:** 表格缺少 aria-label，按钮缺少 focus 样式

## 实施优先级总结

| 优先级 | Sprint | 预估工时 | 核心价值 |
|--------|--------|---------|---------|
| P0 | Sprint 1: 信号焦点+评分解释 | 3-4天 | 解决"信息过载找不到信号" |
| P0 | Sprint 2: 通知告警 | 2-3天 | 解决"必须盯盘"的痛点 |
| P1 | Sprint 3: 历史数据 | 3-4天 | 策略验证、信号回溯 |
| P1 | Sprint 4: 跨维关联 | 2天 | 叙事↔合约打通 |
| P1 | Sprint 5: Detail增强 | 3-4天 | 决策深度 |
| P2 | Sprint 6: 移动端 | 3天 | 扩展使用场景 |
| P2 | Sprint 7: 新手引导 | 1-2天 | 降低理解门槛 |
| P2 | Sprint 8: 数据可靠性 | 1-2天 | 建立信任 |

**总预估: 18-24 天 (按顺序执行)**

建议先从 Sprint 1 + Sprint 2 开始，投入 5-7 天就能让产品体验上一个台阶。
